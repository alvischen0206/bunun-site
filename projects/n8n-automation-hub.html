Case Study

n8n 多功能自動化中樞

從單一大型 workflow 開始，把業務助理、標案查詢、名片掃描、聯絡人搜尋與外部工具整合成自動化中樞。

狀態Building
目前規模約 155 個節點
核心工具n8n、LLM、API
下一步拆分 workflow

發想

一開始用 n8n 做 AI 自動化時，最有效的方法是先把功能串起來，讓它可以跑。等功能越來越多， 同一個 workflow 會逐漸變得難讀、難測、難維護。

這個作品代表的是從「能跑」走向「可維護」的過程。它不是單一小工具，而是一個把不同模組連在一起的自動化中樞。

設計思路

  • 先把 LINE 當作共同入口，避免每個功能都要做一個新的使用介面。
  • 用路由節點與 AI agent 判斷要交給哪個模組。
  • 把記憶、標案、名片、聯絡人與外部工具拆成獨立責任。
  • 未來透過 Execute Workflow 或 API 呼叫，讓每個子流程可以獨立測試。

建議拆分

01
LINE 入口與任務路由
02
記憶系統與上下文管理
03
Tender AI 標案 RAG 查詢
04
名片掃描與 OCR / Vision
05
聯絡人資料庫保存與搜尋
06
天氣、AQI、日期時間等外部工具
07
Stage 3 架構規劃與未來模組 placeholder

技術關鍵字

n8n Workflow Webhook HTTP API Postgres LLM

拆分原則

先備份

完整 workflow 先保存，避免拆壞後回不去。

低風險先拆

先拆 placeholder 與外部工具，再拆核心入口。

每拆必測

每個子流程都要有固定測試問題與回覆格式。
