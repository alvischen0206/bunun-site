Case Study

LINE 公開資訊與一般 AI 查詢

讓使用者不用切換網站或 App，就能在 LINE 裡完成一般 AI 問答、天氣查詢與空氣品質查詢。

狀態Active
入口LINE 文字輸入
核心工具n8n、AI、公開 API
目標日常查詢入口

發想

很多小問題其實不需要打開多個網站查，例如今天會不會下雨、空氣品質如何、某個問題想請 AI 快速整理。 如果這些都能在 LINE 裡完成，使用者就會更願意把 LINE AI 助理當成日常入口。

這個子專案的定位是「低摩擦的資訊查詢」。它不處理公司內部敏感資料，而是先從公開資訊與一般 AI 回覆開始， 讓使用者建立使用習慣。

設計思路

  • 先判斷問題是否屬於一般聊天、天氣查詢或 AQI 查詢。
  • 如果是一般問題，交給 AI 回覆並整理成適合 LINE 閱讀的格式。
  • 如果需要公開資料，透過 n8n 路由到對應 API。
  • 回覆時避免丟原始資料，改成使用者能直接理解的摘要。

目前架構

輸入
使用者在 LINE 輸入一般問題、天氣或空氣品質查詢。
判斷
n8n 判斷是否需要外部工具或直接 AI 回覆。
查詢
呼叫天氣、AQI 或其他公開資訊 API。
整理
把 API 結果轉成簡短、可行動的 LINE 回覆。

技術關鍵字

LINE n8n Weather API AQI AI Reply
