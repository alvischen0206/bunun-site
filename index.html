LINE AI 業務助理

發想：業務工作常常分散在 LINE 對話、客戶紀錄、查詢資料與臨時提醒中。 這個專案希望把 LINE 變成一個熟悉的工作入口，讓日常問題可以直接被分類、查詢與回覆。

設計：以 n8n 接收 LINE webhook，先整理輸入，再交給意圖分析與任務規劃。 系統會依照問題類型路由到聊天、記憶、標案查詢、名片掃描、聯絡人搜尋或工具查詢。

架構：LINE 入口、n8n routing、AI agent、memory controller、外部工具 API 與 LINE 回覆輸出。

Active

Tender AI 標案查詢

發想：標案文件通常頁數多、附件多、格式不一致。人工查找容易漏掉條件、 時程、資格或規格重點，因此需要一個能快速協助查詢與摘要的流程。

設計：把標案 PDF、公告與附件經過文字抽取、OCR、切 chunk、向量化與查詢流程處理， 再由 AI 依照問題回覆重點，降低反覆翻文件的時間。

架構：n8n 問題入口、FastAPI / RAG 查詢服務、PGVector 或向量資料庫、LLM 回答整理。

Building

n8n 多功能自動化中樞

發想：不同功能一開始常被放在同一個 workflow 裡，方便快速測試，但後期會變得難維護。 因此下一步是把大型流程拆成幾個清楚的 n8n JSON。

設計：目前主 workflow 約 155 個節點，已包含入口、意圖分析、記憶、標案 RAG、 名片掃描、聯絡人查詢、天氣 / 空氣品質與第三階段規劃節點。

拆分方向：建議拆成入口路由、記憶系統、標案查詢、名片掃描、聯絡人資料庫、 工具查詢與 Stage 3 規劃七個 workflow。

Experiment

OCR 與名片掃描

發想：業務現場常拿到名片、截圖或紙本資料，如果能快速轉成聯絡人資料， 後續追蹤與整理會省下很多時間。

設計：LINE 上傳圖片後，流程取得圖片、轉 base64、送到 vision model 或 OCR 服務， 再把結果解析成姓名、公司、電話、Email 與備註。

架構：LINE 圖片入口、n8n 圖片處理、Gemini / Qwen vision、資料清理、Postgres 聯絡人資料庫。
